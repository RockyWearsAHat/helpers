## Release Scope

The MCP server is now **Node-free**: agents register a single static Rust binary
(`helpers-native mcp`) that serves every tool — including `search_web` /
`scrape_webpage`, which now drive a real Chrome directly (CDP) instead of Node +
puppeteer. Combined with the v0.4.6 prebuilt binaries, a normal install needs **no
Node and no build toolchain for the tools** — it just works.

## Highlights

- **Node-free MCP server.** `helpers install` now registers `helpers-native mcp`: a
  synchronous JSON-RPC-over-stdio server in Rust (no Node runtime, no warm daemon,
  ~1ms cold start). It serves the native tools plus the web tools and honors the
  live control surface (`helpers disable`, `helpers tool disable`). The Node server
  remains as an unregistered fallback.
- **`search_web` / `scrape_webpage` in Rust via real Chrome.** Automated Google
  search runs in headless Chrome (bypasses anti-bot like the old puppeteer path). On
  a CAPTCHA a **visible Chrome opens for you to solve**, that session is reused so
  later searches run automatically, and any later challenge re-surfaces the window.
  `scrape_webpage` renders JS pages and returns readable text.
- **Clear error states.** If Chrome isn't found, the tools say so and tell you to
  install Google Chrome or set `HELPERS_CHROME_EXECUTABLE`; launch/navigation
  failures return actionable messages so the agent can self-correct.

## Cross-platform

`helpers-native` (now bundling the browser-driven search) is prebuilt for macОS
(universal), Linux x86_64/aarch64/musl, and Windows x86_64/arm64 — all verified in
CI to compile and link with the embedded CDP client (no OpenSSL; CDP is a local
`ws://` socket).

## Notes

- PDF extraction in `scrape_webpage` is not yet ported to the Rust path (returns
  rendered page text).
- The `helpers` CLI itself still runs on Node; only the MCP server (what agents
  spawn for tools) is now Node-free.
