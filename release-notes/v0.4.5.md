## Release Scope

Follow-up to v0.4.4: makes the native crate **warning-clean when built on Windows**
(the path Windows users take, since there are no prebuilt binaries yet). No behavior
change — purely tidies the cross-platform build verified via `x86_64-pc-windows-gnu`.

## Fixes

- **#75 — warning-clean strict_lint on non-Unix.** A Windows source build compiled but
  emitted `dead_code`/`unused_import` warnings, because the VS Code IPC helpers are
  unused when `try_ipc` is the non-unix stub. `ipc_info_path` and the `home` import are
  now `#[cfg(unix)]`, and the `Ipc` enum is `allow(dead_code)` on non-unix (its type is
  referenced by `run()` on every platform; its variants are constructed only on unix).
  Verified zero warnings on both `x86_64-pc-windows-gnu` and the unix host.

## Validation note
