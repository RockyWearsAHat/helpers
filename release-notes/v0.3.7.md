## Release Scope

This release rolls up the unreleased `v0.3.2` through `v0.3.6` development work into one tagged release and adds the final packaging, review-fix, and documentation work completed on the current PR branch.

## Highlights

- Added branch-session workflows for Copilot chats, including isolated worktrees, merge-on-end support, branch-status visibility, and VS Code focus synchronization.
- Shipped an Engram-inspired session-memory system for agents with MCP tools, indexing, retrieval, and VS Code integration.
- Fixed workspace-root resolution so MCP tools target the active VS Code workspace instead of the Helpers install directory.
- Split the MCP executables into thin stable command wrappers plus real `.js` runtime entrypoints and extracted the main subsystems into dedicated `lib/mcp-*.js` modules.
- Added a public GitHub Pages note-search site that indexes the knowledge atlas, Copilot studybase/cache docs, and archived raw research sources.

## Packaging And Distribution

- Added a manifest-driven release layout shared across the shell installer, the macOS package, and package-manager builds.
- Added portable release tarballs with checksum output.
- Added builders for Debian packages, Homebrew formula generation, AUR metadata, npm package tarballs, and VSIX release artifacts.
- Updated the standalone shell installer to install from the packaged archive tree instead of a brittle hand-maintained file fetch list.
- Added optional macOS signing/notarization support in CI and a GitHub release environment bootstrap flow for release secrets and variables.

## VS Code And MCP

- Added language-model discovery, model-tier routing guidance, and stronger diagnostics for subagent model selection.
- Added PDF download/render hardening so non-PDF responses fail fast and rendered page numbering is more reliable.
- Added safeguards against root-level research dump writes and moved legacy raw research files into `research-sources/legacy-root-dumps/`.
- Added public documentation and release assets for the `helpers-server` combined server and the standalone `git-research-mcp` server.

## Search And Knowledge UX

- Added a static Pages search UI under `pages/` plus a site builder in `scripts/build-pages-search-site.js`.
- Added a deployment workflow for the research search site.
- Expanded the test suite to build the Pages search site during CI so deployable assets stay valid.

## Quality And Review Fixes

- Fixed bash portability in the interactive env scrubber UI.
- Fixed cross-platform base64 decoding in env-cache helpers.
- Fixed repo-root handling in history-rewrite helpers so absolute paths no longer break tracked-file discovery.
- Hardened PR shell-lint discovery so filenames with spaces are handled safely.
- Removed the stale hard-coded version from `.github/copilot-instructions.md`.

## Breaking / Operational Changes

- Branch-session binding storage moved to `worktreeBindings.v2`; active sessions from older pre-release builds need to be restarted once after upgrading.
- The supported manual DevOps audit entrypoint remains `/copilot-devops-audit` plus the installed router flow; the release notes and README now treat that path as canonical.
