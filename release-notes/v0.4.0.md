## Release Scope

This release is a full rebrand: the project, its command, binaries, MCP server,
environment variables, and configuration are renamed from **GSH / Git Shell
Helpers** to **Helpers**. There are no behavioral changes — every tool works as
it did in `v0.3.8` — but the rename is intentionally **not backwards
compatible**, so existing installs must re-run the installer.

## What Changed

- **Command:** `gsh` → `helpers`.
- **Binaries:** `gsh-native` → `helpers-native`, `gsh-mcp` → `helpers-mcp`.
- **MCP server:** the stdio server `git-shell-helpers-mcp(.js)` → `helpers-server(.js)`,
  the daemon `git-shell-helpers-mcpd.js` → `helpers-serverd.js`, and the registered
  MCP server name `gsh` → `helpers`.
- **Environment variables:** the `GSH_*` and `GIT_SHELL_HELPERS_*` families →
  `HELPERS_*`.
- **Configuration:** tool state moves from `~/.config/git-shell-helpers-mcp/` to
  `~/.config/helpers-server/`.
- **Installers:** `install-git-shell-helpers` → `install-helpers`, and the
  standalone installer is now `Helpers-Installer.sh`.

The GitHub repository slug (`github-shell-helpers`) and the npm package name are
unchanged, so release URLs, the npm package, and existing clone paths keep
working.

## Upgrading

Because the rename drops back-compat, after upgrading you must re-register the
MCP server with your agent:

```
helpers install     # registers the `helpers` MCP server, recreates config
```

Then reload your agent (Claude Code: restart or `/mcp reconnect`; VS Code:
reload). The old `gsh` MCP registration can be removed safely; it points at a
binary that no longer exists.

## Verification

The rebrand was validated end-to-end before release: `cargo build` + `cargo
test`, the C launcher build, `helpers status`/`doctor`, a stdio MCP handshake
advertising all 20 tools, and the JS regression suite (blackbox, install-health,
roots-workspace, repo-root resolution, project index, tool-doc sync).

## Packaging And Distribution

- Unchanged layout from `v0.3.8`: the manifest-driven release shared across the
  shell installer, the macOS package, and the package-manager builds (Debian,
  Homebrew, AUR, npm, VSIX), with portable checksummed tarballs and CI-built
  artifacts. Artifact names follow the rebrand (`Helpers-Installer-0.4.0.sh`,
  `helpers-0.4.0.vsix`) while slug-based tarball names are preserved.
