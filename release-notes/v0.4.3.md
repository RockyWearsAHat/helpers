## Release Scope

Makes "GSH" / "Git Shell Helpers" work as aliases for Helpers in agent guidance,
and fixes two install issues that kept those updates (and legacy cleanup) from
landing on reinstall.

## Highlights

- **GSH is a recognized alias for Helpers.** The Claude `CLAUDE.md` managed
  block, the `helpers` skill (description + body), the `/helpers` command, and
  the always-on Copilot routing instruction now state that "GSH", "gsh", and
  "Git Shell Helpers" are the former names for Helpers. Asking an agent to "use
  GSH" (or to run a `gsh …` command) now routes to Helpers / `helpers …`. The
  skill's trigger list includes the former names so it activates on either.
- **Reinstall refreshes Copilot guidance.** `helpers install` previously skipped
  any Copilot instruction/skill that already existed (unless `--force`), so
  updates never propagated. It now overwrites the bundled guidance, matching the
  Claude side.
- **Legacy GSH config is pruned on install.** `helpers install` now removes
  pre-rebrand artifacts left by older installs — `~/.claude/commands/gsh.md`,
  `~/.claude/skills/gsh`, and the `~/.copilot/instructions/gsh-*.instructions.md`
  files (including dangling symlinks) — so stale GSH instructions no longer sit
  alongside the Helpers ones.

## Fixes

- Removed rebrand artifacts in the guidance bundle (`Helpers (Helpers)`,
  `Helpers — Helpers`) introduced by the mechanical rename.
