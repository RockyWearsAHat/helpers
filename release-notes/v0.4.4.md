## Release Scope

Fixes the `search_web` CAPTCHA loop (solve once, reuse for every search), and
resolves the four Windows cross-platform issues (#74, #75, #76, #77) that left
clean-machine installs with 0 usable tools. Also lands live MCP tool-list reload.

## Highlights

- **`search_web`: solve the CAPTCHA once, reuse it for all searches.** Previously
  every search that hit a Google CAPTCHA re-prompted you тАФ the interactive solve's
  results were discarded, forcing a headless re-fetch whose different fingerprint
  hit the CAPTCHA again. Now the human-verified browser is kept open and reused for
  every subsequent search/page, the solved query's own results are used directly,
  and verification cookies (incl. `GOOGLE_ABUSE_EXEMPTION`) actually persist.
- **Live tool-list reload.** The MCP server advertises `tools.listChanged` and
  broadcasts `notifications/tools/list_changed`; after `helpers build`/`install`
  the CLI pings connected daemons so already-connected agents pick up new tools
  without restarting the session.

## Fixes

- **`search_web` cookie persistence (was the root of the loop).** The base browser
  profile dir was never created, so the verification cookie write failed with
  `ENOENT` every time and nothing persisted. The base dir is now created before
  writing.
- **#75 тАФ strict_lint Windows build.** `try_ipc()`'s `std::os::unix` socket bridge
  (the crate's only `std::os::unix` use) is now `#[cfg(unix)]` with a
  `#[cfg(not(unix))]` stub; the standalone CLI linters remain the always-available
  primary path. The crate compiles on Windows.
- **#74 тАФ Windows build тЖТ 0 tools.** The C launcher shim's POSIX path is wrapped in
  `#if !defined(_WIN32)` and degrades to the documented Node-stdio fallback on
  Windows, so its compile no longer aborts the build. `lib/mcp-native.js` resolves
  `helpers-native.exe` on Windows.
- **#77 тАФ Windows runtime.** The `helpers` CLI resolves native binaries `.exe`-aware
  and honors `HELPERS_NATIVE_BIN` (built-detection agrees with the loader). Native
  spawns guarantee the Windows system env (`SystemRoot`/`PATH`/`PATHEXT`/тАж) even when
  the server was registered with `"env": {}`, fixing the `exit null` that took down
  the whole tool surface.
- **#76 тАФ onboarding DX.** `helpers doctor` now preflights the build toolchain
  (Rust, C compiler, MinGW-w64 `dlltool` on the Windows GNU host) with per-OS install
  commands. `buildNative` probes the tools before the optional `git-*` symlinks and
  treats symlink failures as non-fatal, so a working build is never aborted. README
  documents the Windows/build prerequisites.
