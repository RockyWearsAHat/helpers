## Release Scope

Bug-fix and polish release. Fixes correctness issues introduced by the v0.4.0
rebrand, completes the v0.4.1 installer/update work, and tidies release assets.

## Fixes

- **Daemon socket scoping (C launcher).** The rebrand changed the env-var prefix
  strings without updating their `strncmp` lengths, so the fast launcher folded
  any `HELP*` variable into the per-workspace socket hash and matched the
  `HELPERS_MCPD_SOCK=` exclusion on the wrong length. The launcher now matches
  the full `HELPERS_` prefix (8) and excludes `HELPERS_MCPD_SOCK=` (18) exactly.
- **Legacy MCP cleanup keeps the right names.** The rebrand rewrote the legacy
  static-registration cleanup lists to `helpers` (the *current* server), so they
  no longer purged the pre-rebrand `gsh` / `git-shell-helpers` entries and risked
  deleting the live one. The VS Code extension and the macOS `.pkg` postinstall
  scripts now remove the legacy `gsh`, `git-shell-helpers`, and
  `git-shell-helpers-mcp` entries and leave `helpers` intact.
- **Packaged installs report the correct version.** The release tarball now ships
  the `VERSION` file next to the CLI, so `helpers update` and the status
  update-hint no longer read `0.0.0` and think an update is always available.
- **`helpers-mcp.c` ships as a source file**, not an executable on `PATH` (it is
  compiled by `helpers build`).
- **Installer surfaces real errors.** The standalone installer no longer hides
  `helpers install` failures behind a generic "no agent detected" message — it
  skips cleanly only when there is genuinely no agent, and reports other failures.
- **AUR `.SRCINFO` attaches to releases.** `upload-artifact@v4` excludes dotfiles
  by default; the release artifact upload now sets `include-hidden-files: true`.
- **macOS postinstall** drops a forbidden heredoc in favor of a single
  `python3 -c` invocation (shell-safety).
