## Release Scope

The `helpers` **control CLI is now a native Rust binary** — the last Node dependency
is gone from the runtime. A normal install downloads **one prebuilt binary for your
platform and just works**: no Node, no toolchain. CI builds the binary for every
platform on version merge and tests a fully Node-free install on macOS, Linux, and
Windows.

## Highlights

- **Node-free CLI.** `helpers` (status, install/uninstall, enable/disable/bypass,
  tool toggles, build, update, grade, index, setup, doctor) is now a Rust module in
  `helpers-native`, invoked busybox-style via the `helpers` symlink. `grade`/`index`/
  `setup` run in-process; the agent config (CLAUDE.md block, skills/commands/agents,
  copilot guidance) is **embedded in the binary**, so `install` is self-contained.
- **Node-free install.** The `curl … | bash` installer (and the shared
  `scripts/fetch-prebuilt.sh` bootstrap) download the prebuilt `helpers-native` for
  your host, symlink the CLIs, and register `helpers-native mcp` — no Node, no source,
  no build toolchain. A source build remains the fallback for unsupported platforms
  (`helpers build --from-source`).
- **Tested everywhere in CI.** The `Install test` workflow builds the binary and runs
  a full Node-free install + verification on **ubuntu, macOS, and windows** (with
  `node` and the toolchain hidden, so a green run proves nothing external is needed).
  It runs on PRs, on pushes to main (install is built+tested on merge), and weekly.

## Notes / follow-up

- The Debian/macOS-pkg/npm package builders still stage the Node tree; converting
  them to the prebuilt-binary bootstrap and deleting the now-dead Node runtime
  (`helpers` script, `helpers-server.js`, `helpers-serverd.js`, `helpers-mcp.c`,
  `lib/mcp-*.js`) is the remaining cleanup. The runtime is already Node-free via the
  binary + curl installer.
- PDF extraction in `scrape_webpage` remains deferred.
