## Release Scope

This release rolls up the unreleased development work since `v0.3.7` into one
tagged release. Its centerpiece is a ground-up rewrite of the Git Shell Helpers
tool surface from Node/AI-coupled scripts to deterministic native Rust, plus a
new static project index and a hardening of workspace resolution so tools always
act on the project the agent is actually working in.

## Highlights

- **Native Rust tool surface.** The deterministic tools — `checkpoint`,
  `strict_lint`, the knowledge subsystem, `cs_lint`/grading, the project index,
  and seven git CLIs (`git-get`, `git-initialize`, `git-upload`, `git-resolve`,
  `git-remerge`, `git-fucked-the-push`, `git-help-i-pushed-an-env`) — are now a
  single `gsh-native` Rust binary. No tool calls an AI model; each is
  standalone, type-safe, and fast.
- **Project index replaces session memory.** A static, near-instant repo map
  (files, symbols, and a ranked reference graph) is exposed via `index_project`,
  `project_map`, and `lookup`, and is shareable across projects as a portable
  `.dxbundle`. This replaces the previous session-memory/`workspace_context`
  system.
- **Agent-agnostic project flows.** `register_workspace_tool` /
  `unregister_workspace_tool` / `list_workspace_tools` let any MCP agent register
  a reusable multi-step flow once and call it by name, scoped to the project via
  `.gsh/tools/manifest.json`.
- **CS2420/CS3500 quality gates.** `cs_lint` emits a prioritized list of concrete
  principle violations with `file:line` fixes, complementing `gsh grade`.
- **Fast startup.** Every entry point routes through a tiny C launcher that
  connects to an auto-managed, per-workspace background daemon, so sessions start
  in ~1ms instead of paying cold Node startup, with an automatic Node fallback.

## Workspace Correctness

- `register_workspace_tool` can no longer write project flows into the GSH
  install repo by accident: a misresolved workspace now fails loudly (overridable
  with `force`) instead of silently contaminating an unrelated repository.
- The MCP server now resolves each session's workspace from the client's `roots`
  capability — the agent's actual open project — rather than the daemon's process
  cwd, and the resolved workspace is threaded into the native binary per call. The
  native schema cache is keyed by workspace so concurrent sessions never see each
  other's project flows. Added an end-to-end roots round-trip regression test.

## Removed

- AI- and editor-coupled features that no longer fit the deterministic,
  agent-agnostic design: the branch-session / worktree-manager workflow, the
  Copilot devops-audit orchestrator, and the legacy session-memory subsystem.

## Packaging And Distribution

- Unchanged from `v0.3.7`: manifest-driven release layout shared across the shell
  installer, the macOS package, and the package-manager builds (Debian, Homebrew,
  AUR, npm, VSIX), with portable checksummed tarballs and CI-built artifacts.
