## Fix

- **`scrape_webpage` no longer hangs.** Current Chrome never replies to the
  graceful `Browser.close` (and `Target.closeTarget`), so the previous
  launch-a-browser-per-call design blocked for the full ~30s idle timeout on
  every teardown — making scrape appear broken and search laggy/spotty. Web
  search and scrape now share **one long-lived headless browser and a single
  reusable tab**, navigating it per call instead of opening/closing browsers or
  tabs. Calls return as soon as results are extracted; the slow close is paid
  only at process exit.

- **Image search works again.** Google retired `tbm=isch` / `/imgres` anchors in
  favour of the `udm=2` Images vertical, so the old extractor matched nothing and
  fell through to a 40s CAPTCHA poll. The extractor now reads `udm=2` result
  thumbnails (title from `alt`, the direct image URL, and a best-effort source
  page), and an empty image grid is reported as "no results" instead of being
  mistaken for a CAPTCHA.

## Improve

- After a solved CAPTCHA the visible window closes off-thread and the freed
  profile lock is retried briefly, so back-to-back searches don't collide.
